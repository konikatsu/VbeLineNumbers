# プロジェクト引き継ぎ

## 目的

VbeLineNumbers は、Microsoft Access の VBE コード画面の左側に行番号を表示する COM アドインです。

VBAコード自体へ行番号を挿入せず、VBEのコード画面に重ねる行番号Overlayとして表示します。

主な対象環境:

- Microsoft Access 2021 64bit
- Microsoft Access 2021 32bit
- .NET Framework 4.8
- Visual Studio / MSBuild

## 現在の状態

- VBE用COMアドインとして動作
- x64 / x86 の Debug / Release ビルドに対応
- 64bit Access は `Addins64`、32bit Access は `Addins` のレジストリパスへ登録
- VBEフォント設定をレジストリから取得
- 行番号OverlayをVBEコード画面左側へ表示
- 縦スクロール、ウィンドウ移動、最大化/元に戻すへ追従
- 背景色と透明度を `VbeLineNumbers.ini` で設定可能
- GitHub Release `v0.1.0` に x64 / x86 のzipを配置

## 重要な設計判断

- 既存の `Guid` と `ProgId` は変更しない。
- VBAコード本文へ行番号は書き込まない。
- 表示はWinFormsのクリック透過Overlayで行う。
- 行高さはフォント計測値を整数ピクセルへ丸める。
- 行番号の縦位置確認は、表示範囲の先頭行と最終行が両方揃うかを基準にする。
- ブレークポイント設定エリアを潰さないよう、行番号Overlayをコード領域へ食い込ませない。
- DLLはGit管理せず、GitHub Releases のzip資産として配布する。

## 主な実装ファイル

- `Connect.cs`
  - COMアドイン接続処理
  - Timer管理
  - VBE CodePane取得
  - Overlay配置
  - RegAsm登録時のAddins/Addins64キー作成
- `LineNumberOverlay.cs`
  - 行番号Overlayフォーム
  - クリック透過
  - 背景色/透明度設定
  - 行番号描画
- `VbeWindowFinder.cs`
  - VBE内のコードウィンドウ探索
  - DPI取得
  - フォント取得対象ウィンドウの判定
- `NativeMethods.cs`
  - Win32 API定義
- `VbeLineNumbers.ini`
  - 背景色と透明度設定
- `README.md`
  - 利用者向けインストール手順

## フォルダー構成

別のPCへ引き継ぐ場合は、まずリポジトリ直下の構成を確認します。

Visual Studioでソリューションとプロジェクトを同じ名前で作成すると、通常は次のような構成になります。

```text
C:\Users\noran\source\repos\VbeLineNumbers
│
├─ VbeLineNumbers.slnx
│
└─ VbeLineNumbers\
   ├─ VbeLineNumbers.csproj
   ├─ Connect.cs
   ├─ README.md
   └─ docs\
```

外側の `VbeLineNumbers` はソリューションフォルダーです。

内側の `VbeLineNumbers\VbeLineNumbers` はC#プロジェクトフォルダーです。

現在のGitリポジトリとCodexの作業フォルダーは、内側のC#プロジェクトフォルダーです。

このプロジェクトの作業フォルダー例:

```text
C:\Users\noran\source\repos\VbeLineNumbers\VbeLineNumbers
```

別PCではユーザー名や配置先が変わるため、以降の手順ではこのフォルダーを「プロジェクトルート」と呼びます。

- `.git/`
  - Git管理情報です。削除しません。
- `.vs/`
  - Visual Studioの一時設定です。Git管理外です。
- `bin/`
  - ビルド出力です。削除しても再ビルドで復元できます。Git管理外です。
- `obj/`
  - 中間生成物です。削除しても再ビルドで復元できます。Git管理外です。
- `dist/`
  - GitHub Release用zipの一時作成場所です。Git管理外です。
- `docs/`
  - 開発、セットアップ、Codex運用のメモです。
- `Properties/`
  - `AssemblyInfo.cs` などのプロジェクト設定です。
- `verification_*.png`
  - 動作確認用スクリーンショットです。Git管理外です。不要なら削除できます。
- `VbeLineNumbers_Test.accdb`
  - 確認用Access DBです。Git管理外です。残しておくと再確認に便利です。
- `VbeLineNumbers_Test.laccdb`
  - Access起動中に作られるロックファイルです。Access終了後は通常消えます。Git管理外です。

過去の `backup_日時` フォルダーは、Git導入前の手動バックアップです。Git運用後は基本的に不要です。

## 配布方針

GitHub Release に以下のzipを置きます。

- `VbeLineNumbers-v0.1.0-x64.zip`
- `VbeLineNumbers-v0.1.0-x86.zip`

zipには以下を含めます。

- `VbeLineNumbers.dll`
- `VbeLineNumbers.ini`
- `README.md`

`bin/`, `obj/`, `dist/` はGit管理しません。

## テストDBの扱い

`VbeLineNumbers_Test.accdb` は確認用DBですが、AccessのバイナリファイルなのでGit管理しません。

目視確認しやすいように、標準モジュールには次のような行を多数入れて使います。

```vb
Rem 1行目
Rem 2行目
Rem 3行目
```

## 既知の注意点

- Accessを起動したままだとDLLが読み直されない。
- Access終了後も `MSACCESS.EXE` が残ることがある。
- `MSACCESS.EXE` を停止する場合は、作業者の確認を取る。
- RegAsm再登録が必要な場合は、実行前に確認を取る。
- 32bit版の確認は、GitHub Releaseからダウンロードしたx86 zipで行う。

## 行番号ずれ確認の基準

途中の数行だけを見ると判断を誤るため、次の2点を必ず確認します。

- 表示範囲の先頭行で、行番号とコード行の縦位置が揃うか
- 表示範囲の最終行で、行番号とコード行の縦位置が揃うか

先頭行が合っていて最終行がずれる場合は、行高さが合っていません。
