# セットアップ手順

このメモは、VbeLineNumbers の開発環境を新しく用意するための手順です。

利用者向けのインストール手順は `README.md` を参照してください。

## Codex

このプロジェクトでは、Codexデスクトップアプリからローカルワークスペースを操作して開発しました。

別PCで同じ作業をする場合は、まずこのプロジェクトをローカルに取得し、そのフォルダーをCodexのワークスペースとして開きます。

例:

```text
C:\Users\<ユーザー名>\source\repos\VbeLineNumbers\VbeLineNumbers
```

Codexで作業する前に確認すること:

- 対象フォルダーをワークスペースとして開けること
- そのワークスペース内のファイルを読み書きできること
- PowerShellを実行できること
- 必要に応じてユーザー承認ダイアログを許可できること
- Access GUI確認を行う場合は、Accessがインストールされていること

Codex自体のインストール方法や最新版の導入手順は、使用しているCodex配布元の案内に従ってください。

Codexにプロジェクトを開かせたら、最初に次を確認します。

```powershell
Get-Location
Get-ChildItem
```

`Get-Location` がプロジェクトルートを指していることを確認してください。

## リポジトリ取得

GitHubから取得する場合:

```powershell
cd C:\Users\<ユーザー名>\source\repos
& "C:\Program Files\Git\cmd\git.exe" clone https://github.com/konikatsu/VbeLineNumbers.git
cd .\VbeLineNumbers\VbeLineNumbers
```

既にフォルダーがある場合は、次で状態を確認します。

```powershell
& "C:\Program Files\Git\cmd\git.exe" status --short --branch
```

## Git for Windows

Git操作には Git for Windows を使います。

確認コマンド:

```powershell
& "C:\Program Files\Git\cmd\git.exe" --version
```

このプロジェクトでは、Codexから明示的に次のGitを呼び出しています。

```text
C:\Program Files\Git\cmd\git.exe
```

## GitHub CLI

GitHub Releaseの確認や資産アップロードには GitHub CLI を使います。

確認コマンド:

```powershell
& "C:\Program Files\GitHub CLI\gh.exe" auth status
```

ログインしていない場合は、次を実行します。

```powershell
& "C:\Program Files\GitHub CLI\gh.exe" auth login
```

ブラウザーに認証画面が出た場合は、表示されたコードをGitHub画面へ入力します。

## winget

wingetを使うには、Windowsの App Installer が必要です。

確認コマンド:

```powershell
winget --version
```

`winget` が見つからない場合は、次を確認します。

```powershell
Get-ChildItem "$env:LOCALAPPDATA\Microsoft\WindowsApps" -Filter winget.exe
$env:Path -split ';' | Select-String 'WindowsApps'
```

`WindowsApps` がPATHにない場合は、ユーザーPATHへ追加します。

```powershell
$userPath = [Environment]::GetEnvironmentVariable("Path", "User")
$windowsApps = "$env:LOCALAPPDATA\Microsoft\WindowsApps"

if ($userPath -notlike "*$windowsApps*") {
    [Environment]::SetEnvironmentVariable(
        "Path",
        "$userPath;$windowsApps",
        "User"
    )
}
```

PATH変更後は、新しいPowerShellを開いて確認します。

## Visual Studio / MSBuild

このプロジェクトは .NET Framework 4.8 のC#プロジェクトです。

Visual Studioで開く場合は、通常は外側のソリューションファイルを開きます。

```text
C:\Users\<ユーザー名>\source\repos\VbeLineNumbers\VbeLineNumbers.slnx
```

ソースコードや `VbeLineNumbers.csproj` は、その下のC#プロジェクトフォルダーにあります。

```text
C:\Users\<ユーザー名>\source\repos\VbeLineNumbers\VbeLineNumbers
```

使用したMSBuild例:

```text
C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\amd64\MSBuild.exe
C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe
```

x64ビルド:

```powershell
& "C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\amd64\MSBuild.exe" `
  VbeLineNumbers.csproj `
  /t:Rebuild `
  /p:Configuration=Release `
  /p:Platform=x64 `
  /p:UseSharedCompilation=false
```

x86ビルド:

```powershell
& "C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe" `
  VbeLineNumbers.csproj `
  /t:Rebuild `
  /p:Configuration=Release `
  /p:Platform=x86 `
  /p:UseSharedCompilation=false
```

## RegAsm登録

このアドインは.NETのCOMアドインなので、`regsvr32` ではなく `RegAsm.exe` を使います。

64bit Access向け:

```powershell
& "$env:WINDIR\Microsoft.NET\Framework64\v4.0.30319\RegAsm.exe" `
  "C:\path\to\VbeLineNumbers.dll" `
  /codebase
```

32bit Access向け:

```powershell
& "$env:WINDIR\Microsoft.NET\Framework\v4.0.30319\RegAsm.exe" `
  "C:\path\to\VbeLineNumbers.dll" `
  /codebase
```

登録解除:

```powershell
& "$env:WINDIR\Microsoft.NET\Framework64\v4.0.30319\RegAsm.exe" `
  "C:\path\to\VbeLineNumbers.dll" `
  /unregister
```

32bit版は `Framework`、64bit版は `Framework64` を使います。

## Release zip作成

Release配布用zipには、DLL、ini、READMEを入れます。

```powershell
$version = "v0.1.0"
$dist = Join-Path (Get-Location) "dist"
New-Item -ItemType Directory -Force -Path $dist | Out-Null
```

作成したzipはGit管理せず、GitHub Releaseへアップロードします。

GitHub Releaseへのアップロード例:

```powershell
& "C:\Program Files\GitHub CLI\gh.exe" release upload v0.1.0 `
  dist\VbeLineNumbers-v0.1.0-x64.zip `
  dist\VbeLineNumbers-v0.1.0-x86.zip `
  --repo konikatsu/VbeLineNumbers `
  --clobber
```

## 確認用DB

確認用DBはワークスペース内の `VbeLineNumbers_Test.accdb` を使います。

ただし、AccessのバイナリファイルなのでGit管理しません。

確認後はAccessを終了します。

```powershell
$access = $null
try { $access = [Runtime.InteropServices.Marshal]::GetActiveObject("Access.Application") } catch {}
if ($access -ne $null) { $access.Quit() }
Start-Sleep -Seconds 2
Get-Process MSACCESS -ErrorAction SilentlyContinue
```

`MSACCESS.EXE` が残る場合、停止する前に作業者へ確認します。
